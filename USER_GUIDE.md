# Claude GUI - 사용자 가이드

## 시작하기

### 필요 조건

- **Claude CLI** 설치 필요 (`claude` 명령어가 터미널에서 실행 가능해야 함)
- macOS 환경

### 실행

```bash
# 개발 모드
wails dev

# 프로덕션 빌드
wails build
# → build/bin/ 에 실행 파일 생성
```

---

## 화면 구성

앱은 **탭 기반** 인터페이스로, 최대 6개의 독립적인 대화 탭을 동시에 운영할 수 있습니다.

각 탭은 다음을 포함합니다:
- **작업 디렉토리** 표시 (탭 헤더에서 클릭하여 변경 가능)
- **대화 영역** (메시지 히스토리 + 실시간 스트리밍 응답)
- **입력창** (텍스트 + 파일 첨부)
- **Plan 모드 토글**

---

## 기본 사용법

### 메시지 보내기

1. 입력창에 메시지 작성
2. `Cmd + Enter` (또는 `Ctrl + Enter`)로 전송

Claude가 응답을 생성하는 동안 실시간으로 텍스트가 표시됩니다.
응답 영역 하단의 타이머로 경과 시간을 확인할 수 있습니다.

### 응답 중지

응답 생성 중 **중지 버튼**을 클릭하면 즉시 생성이 중단됩니다.

### 메시지 재시도

사용자 메시지에 마우스를 올리면 **재시도 버튼**이 나타납니다.
클릭하면 해당 메시지부터 대화를 다시 시작합니다.

> 재시도 시 세션이 초기화되고, 이전 대화 내용은 요약 형태로 Claude에게 전달됩니다.

---

## 세션과 대화 기억

### Claude가 대화를 기억하는 방식

이 앱은 Claude CLI의 **세션 지속성** 기능을 활용합니다.

- **첫 메시지**: 새 세션이 생성됩니다
- **이후 메시지**: 기존 세션을 이어갑니다

Claude는 세션 내에서 **이전에 사용한 도구 결과(파일 읽기, 코드 수정 등)를 모두 기억**합니다.
터미널에서 `claude` 명령어를 직접 사용할 때와 동일한 품질의 응답을 받을 수 있습니다.

### 세션이 초기화되는 경우

다음 상황에서는 세션이 리셋되고 새 세션이 시작됩니다:

| 상황 | 설명 |
|------|------|
| `/clear` 명령 | 대화 기록을 완전히 초기화 |
| 메시지 재시도 | 해당 지점부터 새로 시작 (이전 내용은 요약 전달) |
| 작업 디렉토리 변경 | 새 디렉토리에 맞는 새 세션 필요 |
| 탭 닫기 | 세션 자원 해제 |

## 파일 첨부

### 방법

- **파일 선택 버튼**: 입력창 옆의 버튼으로 파일 탐색기 열기
- **드래그 앤 드롭**: 파일을 입력창으로 직접 끌어다 놓기

### 지원 파일

- **코드 파일**: `.go`, `.js`, `.ts`, `.py`, `.java`, `.kt`, `.swift`, `.rs`, `.rb`, `.php`, `.c`, `.cpp` 등
- **이미지 파일**: `.jpg`, `.png`, `.gif`, `.webp` (미리보기 제공)
- **문서 파일**: `.md`, `.txt`, `.json`, `.yaml`, `.toml`, `.sql` 등
- 파일당 최대 **10MB**

## 슬래시 명령어

입력창에 `/`를 입력하면 사용 가능한 명령어 목록이 팝업으로 표시됩니다.

| 명령어 | 설명 |
|--------|------|
| `/help` | 사용 가능한 명령어 목록 표시 |
| `/clear` | 대화 기록 초기화 (세션도 함께 리셋) |
| `/usage` | 토큰 사용량 추정치 표시 |
| `/model` | 현재 모델 확인 / 변경 (인터랙티브 선택) |
| `/model <이름>` | 지정한 모델로 직접 변경 |
| `/snippet <경로> [시작-끝]` | 코드 파일의 특정 라인을 메시지에 첨부 |
| `/apply` | Plan 모드 분석 결과를 실행 모드로 전환 |

### 예시

```
/model claude-opus-4-20250514
/snippet app.go 10-30
/clear
```

## Plan 모드

### 개요

Plan 모드는 **읽기 전용** 모드로, Claude가 코드를 수정하지 않고 분석만 수행합니다.

- 탭 헤더의 **Plan 모드 토글**로 활성화/비활성화
- Plan 모드에서 Claude는 `Read`, `Glob`, `Grep`, `WebSearch`, `WebFetch` 도구만 사용 가능
- 파일 수정, 쓰기, 명령어 실행 도구는 **물리적으로 차단**됨

### 사용 흐름

1. Plan 모드 활성화
2. 구현하려는 기능/수정 사항을 설명
3. Claude가 코드베이스를 분석하고 **상세 구현 계획** 제시
4. 계획이 만족스러우면 `/apply` 명령으로 실행 모드 전환
5. Plan 모드가 해제되고, Claude가 계획을 실행

> Plan 모드의 분석 결과는 세션에 유지되므로, `/apply` 실행 시 Claude가 자신이 세운 계획을 기억합니다.

## 작업 디렉토리

각 탭은 독립적인 **작업 디렉토리**를 가집니다.

- 탭 헤더의 디렉토리 경로를 클릭하여 변경
- 경로 입력 시 **자동완성** 지원 (Tab 키)
- `~`로 시작하면 홈 디렉토리로 확장

> 작업 디렉토리를 변경하면 세션이 초기화됩니다. Claude CLI 세션은 디렉토리 기반으로 동작하기 때문입니다.

## 탭 관리

| 동작 | 방법 |
|------|------|
| 새 탭 추가 | 상단의 "+" 버튼 (최대 6개) |
| 탭 닫기 | 탭 헤더의 X 버튼 |
| 탭 이동 | 탭을 드래그하여 위치 이동 |
| 탭 분할 | 탭을 다른 탭 위로 드래그하여 상/하/좌/우 분할 |
| 탭 교환 | 탭을 다른 탭 중앙으로 드래그하여 위치 교환 |
| 레이아웃 정리 | "정리" 버튼으로 그리드 재배치 |

## 모델 선택

사용 가능한 모델:

| 모델 | 특징 |
|------|------|
| `claude-sonnet-4-20250514` | 기본값. 속도와 품질의 균형 |
| `claude-haiku-4-20250414` | 빠른 응답. 간단한 작업에 적합 |

변경 방법: `/model` 명령어 또는 `/model claude-opus-4-20250514`

## 응답 확대 보기

긴 응답의 경우 **확대 보기** 버튼을 클릭하면 브라우저에서 전체 화면으로 열립니다.

## 문제 해결

### "Claude service not initialized"

Claude CLI가 설치되지 않았거나 PATH에서 찾을 수 없습니다.

```bash
# Claude CLI 설치 확인
which claude

# 버전 확인
claude --version
```

### 응답이 생성되지 않음

1. 터미널 로그에서 `[CLIWrapper]` 접두사의 에러 메시지 확인
2. `/clear`로 대화 초기화 후 재시도
3. 탭을 닫고 새 탭에서 시도

### 세션 관련 문제

세션 이어가기(`--resume`)가 실패하면 자동으로 새 세션으로 재시도됩니다.
수동으로 해결하려면 `/clear` 명령으로 세션을 초기화하세요.

### 응답 대기 시간 초과

10분간 출력이 없으면 프로세스가 자동 종료됩니다.
이 경우 메시지를 다시 보내주세요.
